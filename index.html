Espectativa vs. Realidad
Vagrant

“Herramienta que permite construir entornos de desarrollos, isolados en maquinas virtuales y orientado a la automatización.”

Vagrant

  • Creado por Mitchell Hashimoto.

  • Utiliza tecnologías standards:

    • Soporte para VirtualBox, VMware, AWS...

    • Provision en shell scripts, Chef y Puppet.

  • Agnostico al lenguaje de programacion/framework usado.

  • Hoy en dia, bastante popular.

  • Buena documentación.

  • Soporte de la comunidad.

Vargant
Vagrant

Si sos Dev:

  • Podés tener entornos consistentes (en una VM) y aislados de lo que tengas instalado en tu computadora (navegadores, IDEs, debuggers, etc.).

  • Todos en un equipo comparten las mismas configuraciones de entorno de trabajo.

  • Podés setear un entorno con solo usar unos comandos.

  • Si ya lo tiene seteado, también puede actualizar las configuaciones en poco tiempo.

  • Totalmente independiente del Sist. Op. que tengas en tu computadora (bueno, casi).

Vagrant

Si sos SysAdmin:

  • Podés tener disponibles uno o mas entornos isolados con sus propias configuraciones.

  • Amplia gama de herramientas, scripts shell, Chef cookboks, maquinas virtuales y mas en tus servidores locales o en Amazon. Todos en un equipo tienen el mismo entorno de trabajo.

  • Una buena herramienta para organizar y unificar el trabajo.

Vagrant

Si sos Diseñador:

  • Permite abstraerlo del setup de su entorno.

  • Disminuye el impacto cada vez que hay cambios en la configuración.

Vagrant

Ventajas:

  • Customizable.

  • Set de comandos simples.

  • Permite tener entornos muy similares al de Producción (en todo lo posible).

  • Posibilidad de tener una VM por proyecto/aplicación, cada una con distintas configuraciones.

  • Flexibilidad a la hora de elegir con que herramienta automatizar (provisioner).

  • No hay conflicto con Git, SVN o Mercurial.

  • Entornos portables y reusables.

Vagrant

Desventajas:

  • Uso de VM => consume recursos (a la maquina host).

  • Es multiplataforma pero hay algunos problemas (NFS problemas peculiares con Microsoft)

  • La gente con poca experiencia en setear entornos, tienen menos chances de aprender con Vagrant.

Vagrant

Algunos conceptos:

  • Box: imagen de Sist. Operativo a partir del cual pueden generar VMs.

    • vagrant box add precise32 http://files.vagrantup.com/precise32.box

  • Carpetas sincronizadas: Vagrant sincroniza automáticamente la carpeta donde esta el Vagrantfile con una carpeta en la VM, por default: /vagrant

  • Provisioning: (o aprovisionamiento) herramienta que permite instalar y configurar servidores en forma automatica.

Vagrant

Instalación:

  • Instalar un software de virtualización (recomendable: VirtualBox).

  • Bajar instalador desde: http://www.vagrantup.com/downloads.html


    • En Windows, instalan con el archivo vagrant_1.7.2.msi


    • En Debian/Ubuntu, ejecutan:
      sudo dpkg -i vagrant_1.7.2_x86_64.deb


Vagrant

Comandos mas usados:

  • vagrant init: inicializa un entorno Vagrant.

  • vagrant up: la 1° vez crea la VM, la inicializa y la configura (en base al Vagrantfile). El resto de las veces solo hace las últimas 2 cosas.

  • vagrant provision: (en caso de haber uno) ejecuta y configura el provisioner en la VM correspondiente.

  • vagrant halt: apaga la VM

  • vagrant reload: lo mismo a hacer vagrant halt + vagrant up

  • vagrant ssh: hace un ssh a la correspondiente VM.

Vagrant

Workflow de comandos:

  • vagrant init: solo se lo ejecuta una sola vez.

  • vagrant up: inicializa la VM

  • vagrant provision: en caso de tener alguno, se lo usa cuando hubo cambios en el mismo.

  • vagrant ssh: en caso de necesitar, se accede a la VM.

  • ...

  • Desarrollo o prueba a realizar.

  • ...

  • vagrant halt: apagamos la VM.
